## 26.04.07. MVP Pattern 응용 과제
1. 구현 예제 : 뽑기 시스템. 1 or 5~
2. 규칙 :
   1) 뽑기 재화(유료든 무료든)가 충분한가 확인
   2) 플레이어는 단수, 복수 중 하나를 고를 수 있다.
   3) 단수일 경우, 하나. 복수일 경우 지정한 횟수만큼(시스템적으로) 확률적으로 뽑는다.
   4) 뽑기 결과를 출력하고, 남은 재화를 출력한다.

-> 그렇다면 표시해야 할 부분.
   1. 플레이어의 재화 -> 뽑기 전, 뽑기 후 재화 변동이 일어날 것.
   2. 뽑기 결과 화면 -> 단수, 복수에 따라 달리 출력될 것.
   3. 뽑기 단수, 복수에 따라 무엇을 선택할 것인지에 대한 인터페이스 제작.


주의:
   1) Model - 데이터를 연산하는 책임, View - 데이터를 표시하는 책임,
   2) Model과 View를 완전히 분리 
   3) Observer Pattern Base -
           Model과 View는 각각의 존재와 Presenter를 인지하지 못하며 이벤트만 발생시킨다.
           Presenter는 각각의 Model, View의 이벤트와 Public Method만 인지한다.
              -> 이벤트가 발생할때마다 이벤트 - Method를 바인딩하여 실행한다.

   4) 수동적 뷰와 감시자 컨트롤러로 나뉜다.
      > 수동적 뷰 : 사용자에게 데이터를 보여주고, 입력을 받는 껍데기.
                    독자적으로 판단하고 실행하는 로직이 전혀 없는 인터페이스
      > 감시자 컨트롤러 : 단순한 데이터 동기화는 View 자체에게 부여하며
                          View에 대한 복잡한 처리만 Controller가 개입한다.(= Model 정보 의존)
